# k0s Ansible Playbook — Dev/Prod Split

Installs k0s (single-node dev, or multi-node prod) with Cilium CNI, kube-proxy
replacement, and environment-specific storage. Dev uses Longhorn; prod uses the
AWS EBS CSI driver with native `gp3` volumes. Tested on Arch Linux (bare metal,
dev) and Ubuntu 24.04 (AWS EC2, prod).

## Structure

```
.
├── site.yml              # Playbook: 3 plays (controller, kernel tuning, worker join)
├── inventory.ini         # Dev host (localhost). Prod: generated by scripts/generate-inventory.sh
├── requirements.yml      # Ansible collection dependencies (ansible.posix)
└── group_vars/
    └── all.yml           # Variables — review before running
```

## Before running

**0. Install Ansible collection dependencies**

```bash
ansible-galaxy collection install -r requirements.yml
```

Required by the `ansible.posix.sysctl` module used for kernel tuning on prod sandbox nodes.

**1. Edit `inventory.ini`** (dev only — prod inventory is auto-generated by `scripts/generate-inventory.sh`)

Local dev (Arch):

```ini
[k0s_controller]
localhost ansible_connection=local
```

**2. Review `group_vars/all.yml`**

Key values to confirm (defaults are tuned for local dev / single node):

- `env: "dev"` — override with `-e env=prod` for production
- `pod_cidr: "172.16.0.0/16"` — must not overlap with your node network or VPC CIDR
- `service_cidr: "10.96.0.0/12"` — must not overlap with your VPC CIDR
- `k0s_version: "v1.35.4+k0s.0"`
- `cilium_version: "1.19.4"`
- `helm_version: "v4.1.4"`
- `sandbox_isolated_cpus: "2-7"` — prod sandbox kernel isolation range

## Running

```bash
# Install collection deps first (once)
ansible-galaxy collection install -r requirements.yml

# Dev: dry run
ansible-playbook -i inventory.ini site.yml -e env=dev --check

# Dev: full run
ansible-playbook -i inventory.ini site.yml -e env=dev

# Prod: generate inventory from Terraform outputs, then run
bash scripts/generate-inventory.sh
ansible-playbook -i inventory-prod.ini site.yml -e env=prod
```

## Notes

- **kubectl interaction**: After the playbook completes, a kubeconfig is placed in `/home/<user>/.kube/config`. To interact with the cluster, always use:
  ```bash
  KUBECONFIG=~/.kube/config k0s kubectl <command>
  ```
- Dev uses `--enable-worker --no-taints` (single-node). Prod runs a separate controller, with platform and sandbox workers joining in Play 3.
- Pod CIDR `172.16.0.0/16` is chosen to avoid conflict with AWS VPC `10.0.0.0/16` and typical home/lab `192.168.x.x` ranges.
- Cilium uses VXLAN tunnel mode for cloud agnosticism — works on bare metal and AWS without VPC route table changes.
- **Dev storage**: Longhorn (software-defined, local disk). **Prod storage**: AWS EBS CSI driver with native `gp3` — no software replication overhead on top of EBS.
- Prod sandbox nodes get kernel isolation (`isolcpus`, `nohz_full`, `rcu_nocbs`) plus `irqbalance` disabled, performance CPU governor, and `vm.swappiness=0` for microsecond-level determinism.
- The playbook is idempotent — safe to re-run.
