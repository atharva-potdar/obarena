# Submission API

## Overview

HTTP service accepting contestant code uploads. Stores artifacts to SeaweedFS,
publishes lifecycle events to Redpanda. Single responsibility — no auth, no
status tracking, no build logic.

## Endpoints

### Submit Code

POST /submissions
Content-Type: multipart/form-data

Fields:

- `source`    file      tar.gz of contestant source code (entire project with vendored dependencies)
- `language`  string    "cpp" | "rust" | "go"
- `team_name` string    display name for leaderboard

Response 202:
{
  "submission_id": "uuid"
}

Response 400:
{
  "error": "unsupported language" | "file too large" | "invalid archive"
}

Response 500:
{
  "error": "internal error"
}

### Health Check

GET /healthz

Response 200:
{
  "status": "ok"
}

## Event Schema

Topic: submission.lifecycle
Key:   submission_id (ensures ordered processing per submission)

submission.created
{
  "event":          "submission.created",
  "submission_id":  "uuid",
  "language":       "cpp" | "rust" | "go",
  "team_name":      "string",
  "artifact_path":  "submissions/{submission_id}.tar.gz",
  "created_at":     1234567890  // unix nanoseconds
}

## Configuration

SEAWEEDFS_ENDPOINT    SeaweedFS S3 endpoint
                      default: <http://seaweedfs.platform.svc.cluster.local:8333>
REDPANDA_BROKERS      comma-separated broker list
                      default: redpanda.platform.svc.cluster.local:9092
PORT                  listening port
                      default: 8080
MAX_UPLOAD_SIZE_MB    max upload size in MB
                      default: 50

## Constraints

- Only tar.gz archives accepted. The file will be the entire project, with vendored dependencies.
- Supported languages: cpp, rust, go
- Max upload size: 50MB (configurable)
- Submission ID is a UUID v4 generated by this service
- SeaweedFS bucket "submissions" must exist before service starts
- No deduplication — each upload gets a new submission ID

## Dependencies

- github.com/twmb/franz-go       Redpanda producer
- github.com/aws/aws-sdk-go-v2   SeaweedFS S3 client
- github.com/google/uuid         UUID generation
